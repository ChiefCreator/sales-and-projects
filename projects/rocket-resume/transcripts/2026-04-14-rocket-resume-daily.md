### Запросы клиента и направление проекта

- Клиент попросил систему для хранения проектной информации (knowledge base) в AI-native формате
- Обсуждалась необходимость полноценной системы для управления бэклогом с иерархией задач (как в Jira с UserStory, Feature, Epic), так как Trello не подходит для такого уровня детализации
- Команда планирует долгосрочное сотрудничество и готова настроить нормальную систему управления проектом

### Прогресс по технической реализации

- Страницы успешно собираются и задеплоены, хотя были проблемы с билдами
- Есть проблемы с кэшированием - страницы грузят закэшированные версии
- Деплой в main не работает из-за отсутствия HyperDrive, используется ручной деплой
- Навигация работает корректно
- Аутентификация между их сайтом и новым не синхронизируется - куки кодируются по-разному
- Код навигации требует рефакторинга, но пока находится в одном месте

### Стратегия первого релиза

- Решено фокусироваться на low-traffic страницах для первого релиза как тест новой инфраструктуры
- Customer support упомянут как пример страницы с низким трафиком
- Сложные секции (PDF рендер, builder templates) будут отложены, чтобы не тратить время на функциональность, которую могут переделать в дизайне
- Простые контентные секции будут добавлены для job titles и industry страниц
- Лендинговые страницы (contact, customer support, terms & conditions) планируются к переносу для первого релиза

### Структура страниц и CMS

- У клиента есть два основных типа страниц: job titles и industry (визуально похожи или одинаковые)
- Job title страницы используют template-based подход - одна страница-template с динамической логикой для генерации множества страниц с разными слагами
- Denis создал систему с template literals для shuffle секций и динамического контента
- Сам клиент не полностью помнит, как реализована логика job titles
- Landing pages отличаются от template-based страниц - это обычные страницы в CMS, которым дают слаг и добавляют секции
- Для лендинговых страниц не хватает некоторых секций для полного переноса

### Сложные секции для дальнейшей проработки

- Секции с PDF рендерингом требуют значительной работы
- Секции с builder templates, которые ведут напрямую в builder
- Нужно определить, откуда берутся ссылки для некоторых секций и нужно ли их добавлять в CMS

### Обсуждение инструментов разработки (Claude AI)

- Обсуждался новый LSP (Language Server Protocol) в Claude, который использует нативные функции языка вместо grep для поиска, что значительно ускоряет работу
- Проблема с лимитами токенов - Eugene купил второй личный аккаунт для продолжения работы
- Идея использовать $200 аккаунты вместо Team plan для большего лимита токенов
- Обсуждался инструмент для создания knowledge base, который может сэкономить до 90% токенов на research и базовые задачи редактирования кода
- Knowledge base работает через cron job для синхронизации, low-level изменения Claude делает в live режиме
- Рассматривается возможность поднять Gemini как fallback решение

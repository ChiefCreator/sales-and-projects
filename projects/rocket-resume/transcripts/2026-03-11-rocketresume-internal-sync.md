### Обзор системы генерации лендингов

Денис представил реализованную систему генерации лендингов с двумя основными типами страниц: по индустриям (healthcare, airline и др.) и по job titles.  Система включает специальные суффиксы для SEO-оптимизации, которые влияют на генерацию контента страниц. 

**Основные возможности:**

- Шаблонизатор с переменными, которые автоматически заменяются в зависимости от индустрии и job title
- Вариации контента — некоторые элементы меняются словами, другие полностью переформулируются
- Ротация секций — секции появляются в случайном порядке на разных страницах
- Поддержка canonical URL для суффиксов (один canonical на комбинацию индустрия + job title, остальные суффиксы ссылаются на него)
- Таргетированное переопределение секций для конкретных индустрий или job titles

### Структура CMS

**Организация контента:**

- Глобальные шаблоны для индустрий и job titles, которые наследуются всеми страницами
- Коллекции слов (разные формы глаголов, предлогов) для использования в шаблонах
- Cheat sheet со всеми доступными переменными
- Поддержка генерации отдельных пользовательских страниц (например, Home)

**Поддержка тенантов:**

- Конфиг для каждого тенанта определяет, для каких индустрий и job titles генерировать страницы
- Планируется добавить таб с CSS-кастомизацией
- Обработка через middleware на фронтенде
- Поддержка локализации

### Технические детали и проблемы

**CloudFlare деплоймент:**

- Все развернуто на CloudFlare, работает стабильно без задержек и глюков
- Настройка была сложной — пришлось вручную настраивать базы данных, кэширование
- Кэш работает, draft mode работает

**Проблемы с деплоем и миграциями:**

- Автоматический деплой не настроен, так как нельзя дать доступ к репозиторию, а у клиента не оплачен план на Worker
- Серьезные проблемы с миграциями базы данных: нет поддержки транзакционности, все create statements выполняются как отдельные SQL-запросы
- Невозможно запустить миграции на remote базу через CLI, приходится создавать отдельный SQL-файл
- Надежда на то, что Xata решит эти проблемы

### Сравнение с текущим решением клиента

Реализация отражает то, что уже есть у клиента, возможно, даже немного лучше.  User-facing функционал работает нормально без заметных проблем.  

**Неожиданные требования:**

- Вся система вариаций слов и суффиксов не обсуждалась ранее в скоупе
- Изначально казалось, что нужна простая генерация "одна страница на один job title", но оказалось это целая система

### Статус работ

**Сделано:**

- Основная система генерации лендингов
- Шаблонизация и вариации контента
- Поддержка тенантов

- Базовый UI (минимальные стили, несколько секций)

**В работе/планируется:**

- BetterOS интеграция — ждем требований, возможно можно использовать open-source плагин
- Resume builder — следующий этап после согласования текущего функционала
- Skills — еще не разобран

### Проблемы с дизайном

Дизайн пока плохо продвигается.  Команда дизайна начала перерабатывать воронки (funnels) клиента с нуля, предлагая новые бизнес-решения, хотя ожидалось, что они просто улучшат существующий дизайн и контент без полной переработки структуры.   Это не то направление работы, которое требовалось.

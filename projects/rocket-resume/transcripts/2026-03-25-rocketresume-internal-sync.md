Summary

### Задачи и следующие шаги

- [ ]  Денис показать текущий прогресс по MVP на звонке с клиентом
- [ ]  Запросить у клиента конкретные приоритетные задачи для оставшихся 2.5 дней работы Дениса
- [ ]  Клиент должен подготовить структурированный бэклог к среде следующей недели для расширенной команды
- [ ]  Денис подключиться на один звонок на следующей неделе для онбординга Павла
- [ ]  Уточнить у клиента детальные требования по аналитике и трекингу
- [ ]  Задать вопросы по всем абстрактным задачам из роадмапы клиента

### Статус MVP

- Денис отработал 65 часов из ~80 запланированных, осталось примерно 2.5 дня
- Сделано: вариации лендингов на базе нескольких уникальных компонентов
- Реализовано: BetterOAuth (базовая версия, работает на отдельной базе с отдельными юзерами)
- Реализовано: плавный GoOnLive перекат на prod (на демо-страницах, без полного внедрения в CMS)
- Показан Search функционал
- Остались небольшие фиксы, которые можно сделать за пару часов

### Планы по команде

- Денис завершает MVP и уходит на паузу в апреле
- В апреле на проект заходят Женя и Павел
- В мае Денис вернется третьим человеком, возможно добавится еще один разработчик
- Жека на следующей неделе в отпуске
- Анастасия подключается как PM для координации с клиентом

### Проблемы и вызовы

- Отсутствие четких требований и приоритетов от клиента
- Непонятна общая картина взаимодействия с командой клиента (Иван, Габриэль)
- Дизайны еще не готовы, команда клиента выясняет палитру цветов и выбирает концепты
- Нет ясности по миграции с Contentful
- Требования от Стива слишком абстрактные, нужна детализация

### Обсуждаемые фичи и задачи

- Аналитика: клиент хочет lightweight решение с Mixpanel, опасается влияния на page load
- Перестройка билдера
- Перестройка чек-аута
- Работа с блогом и CMS
- Search APIs (требует уточнения)
- Backend improvements и улучшение API
- Внедрение финальных дизайнов (когда будут готовы)

### Договоренности по процессу работы

- Формат работы на MVP был неструктурированным, для расширенной команды нужен четкий процесс
- Клиент должен документировать и приоритизировать задачи
- Для абстрактных задач предлагается создавать technical proposals с детальными решениями на согласование
- Рассматривается опция зарезервировать оставшиеся часы Дениса, если клиент не готов с задачами

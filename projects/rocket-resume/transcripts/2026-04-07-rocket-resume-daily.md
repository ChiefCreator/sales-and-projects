### Работа над хедером

- Паша завершил работу над хедером, перенес всю логику, но не может открыть PR из-за ограничений на публичный репозиторий

- **Важное изменение в требованиях**: хедер должен быть в точности как у клиента, а не улучшенная версия
- Причина - пользователь не должен замечать разницу между двумя серверами при навигации
- HTML и JavaScript bundle текущего хедера должен подставляться на место нового
- Модалка логина тоже должна выглядеть идентично
- Данные можно брать из CMS, сделать заглушки, чтобы не триггерить их endpoints

### Архитектура прокси и роутинга

- Текущее состояние: в монорепе лежит MVP и роутер в GitNation Accounts на Cloudflare, который редиректит трафик на разные сервера
- Денис создал демо-проект на Vercel, который проксирует запросы либо на MVP, либо на demo в зависимости от пути
- Умный роутер переписывает логику навигации, конфиг лежит в KV
- **План**: убрать demo и сделать прокси в rocket-режиме на их Cloudflare
- Прокси будет редиректить трафик либо на Vercel, либо на Cloudflare клиента
- Тестирование будет через [worker.dev](http://worker.dev) до привязки домена
- После успешного тестирования можно привязывать домен

### Миграция на Cloudflare клиента

- Нужно перенести два [workers.page](http://workers.page) на аккаунт клиента со всеми binding'ами
- Все AC базы перечислены: база, self-reference, R2
- В wrangler файле нужно изменить ID аккаунта и раскомментировать строку
- CloudFlare должен зарезолвить все binding'и с уникальными ID
- Существующие данные в базе нужно дампнуть и закинуть в новую базу
- Neon остается тот же, не переносится
- Hyperdrive (connection pool на CloudFlare) нужно настроить с connection string
- Кеши не критичны для переноса

### Вопросы логина

- Логин будет работать через endpoint клиента, их базу с юзерами
- Планируется скопировать логику логина с их старого сайта
- Когда домен будет привязан к прокси, браузер начнет шарить куки
- До привязки домена будет момент с отдельной работой логин-формы
- Варианты: полностью скопировать логику или делать редирект запросов

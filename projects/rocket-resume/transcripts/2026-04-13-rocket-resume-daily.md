### Система управления знаниями на базе Obsidian

- Pavel начал настройку Obsidian Storage с использованием архитектуры от Карпатого
- Архитектура получила большой отклик в Twitter и Reddit
- Уже экспортированы все чаты из Claude, проставлены теги через Gemma-4 (Google)
- Система автоматически разбивает чаты на основные тезисы и концепты, строит графы зависимостей
- Планируется загрузка всех транскриптов митингов и информации по проектам из Notion
- Настроен автоэкспорт завершенных чатов в базу с еженедельным резюмированием через cronjob (по субботам)
- Система работает через индекс небольшого размера, который загружается при старте чата
- Через локальную MCP можно обращаться к memory points
- Obsidian имеет плагин для автоматического пуша в Git

### Преимущества новой архитектуры

- Отличие от RAG: единая точка входа с большим контекстом, из которого нейросеть сама вытягивает нужное
- Возможность масштабирования на уровень команды (15+ активных пользователей)
- Автоматическая синхронизация и обновление базы через GitHub actions
- Использование Gemma-4 вместо дорогих моделей для обработки (весит 20GB vs 400GB+ у других open-source моделей)

### Применение для компании Focus Reactive

- Необходимость knowledge base на уровне компании для всех проектов
- В Notion уже есть много информации: транскрипты звонков, sales-коммуникации, проектные данные
- Система поможет с написанием статей, подготовкой к sales-звонкам
- Текущий поиск в Notion медленный и требует много ручной работы с промптами
- Pavel может заниматься этим как внутренним проектом компании
- Необходимо дописать скиллы под конкретные потребности компании
- Все должно быть на английском для использования в компании
